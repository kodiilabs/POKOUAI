## Project Overview
**PokouAI** is an offline-first, multilingual AI crop disease advisor for West African smallholder farmers. Built for the Gemma 4 Good Hackathon (deadline: May 18, 2026).

- **Primary crop**: Cocoa (Ivory Coast)
- **Primary target**: Farmers on entry-level Android/iOS with 2G or no internet
- **Core constraint**: 100% offline inference via llama.cpp

## Critical Constraints

### Offline-First Design
- **Rule**: Every feature must work with zero internet. Cloud is enhancement only.
- **Test**: Always verify app launches and performs diagnosis in airplane mode.
- **Storage**: GGUF model bundled in app (~2.5â€“3.0 GB) or lazy-downloaded on first launch.

### Multilingual Support
- **Four languages required for v1**: French, Dioula, BaoulĂ©, English
- **Dioula & BaoulĂ©**: Low-resource languages â€” use native speaker review
- **Response format**: All disease responses must include (1) disease name, (2) treatment steps, (3) prevention advice in all 4 languages
- **UI strings**: Use i18n keys, never hardcode text

### Hardware Constraints
- **Target device**: 2â€“3 GB RAM entry-level Android â€” this is what determines the default model choice
- **Primary model**: Gemma 4 **E2B** Q4_K_M (~1.5 GB, runs in ~2 GB RAM). Chosen because E4B Q4_K_M (~2.8 GB) would leave no headroom for the OS and camera pipeline on the target device.
- **Premium model**: Gemma 4 E4B Q4_K_M for phones with â‰Ą4 GB RAM (optional download in Settings)
- **Inference time goal**: <15 seconds per image on emulator (faster on real device)
- **Image size**: Max 1024px (limit memory usage + transmission)

### 3-tier routing (Cactus/Ollama prize strategy)
- **Tier 1 â€” local**: Gemma 4 E2B via llama.cpp on the phone. Always available. Targets llama.cpp Prize.
- **Tier 2 â€” hub**: Gemma 4 27B via Ollama on a cooperative laptop over local WiFi (no internet). Configurable hub URL in Settings â†’ Hub. Targets Ollama Prize and powers the extension-worker group-mode teaching flow.
- **Tier 3 â€” cloud**: Gemma 4 27B hosted for anonymized sync + outbreak map when internet is present.
- Routing logic: `app/src/services/InferenceRouter.ts`. Probes hub + internet in parallel, picks highest available tier, falls back on error. Tier recorded on every diagnosis and surfaced on Result screen.

### Education layer (Future-of-Education prize)
- Four modes: Learn (cause explanation after diagnosis), Prevention Calendar (seasonal), Quiz (spaced repetition on recent diagnoses), Group Mode (extension worker teaching UI).
- Group mode skips personal-log persistence; quiz/learn read from `services/knowledge.ts` which sources from `cocoa_diseases.json`.

### Scientific farming loop (Addendum v2)
- A "loop" = initial diagnosis â†’ hypothesis (5 s tap on Result screen, optional voice memo) â†’ +7 day local notification â†’ comparative follow-up diagnosis & outcome capture â†’ 1-line lesson â†’ Farm Intelligence Log entry.
- Data model: `loops` table (`services/db.ts`) joins initial + follow-up diagnoses, holds `hypothesis_category`, `hypothesis_audio_uri` (voice memo path), `comparison_response` (raw model output). CRUD in `services/loops.ts`. Notifications in `services/notifications.ts` (local only, no internet required to schedule or fire). Voice in `services/voice.ts` (expo-av).
- Inline UI: `components/HypothesisCard.tsx` on `ResultScreen` â€” 4 tappable theories + đźŽ™ voice-memo button (record/stop, plays back when set).
- Day-7 flow: `FollowUpScreen` calls `routeComparison(beforeUri, afterUri, lang, diseaseName)`. Hub + cloud receive both images; local llama.cpp falls back to single-image with comparison-aware prompt. Result is saved on `loops.comparison_response`.
- Curated record: `FarmIntelligenceLogScreen` shows pending + completed loops with lessons; pending rows that are past `scheduled_for` get a red border. Audio playback pill on cards with a voice memo.
- Permission backstop: `HomeScreen` shows a red overdue banner when any pending loop is past `scheduled_for`, deep-linking the oldest. This is the failsafe when notification permission was denied or the user just doesn't see notifications.
- Notification deep-link: `App.tsx` listens on `Notifications.addNotificationResponseReceivedListener` and routes `kind:'followup'` taps directly to `FollowUp(loopId)`.
- This is the Future-of-Education positioning â€” not "the app teaches science," but "the app structures the scientific method the farmer is already practicing." See `docs/PokouAI_Addendum_v2.md` for the framing.

## Code Style & Standards

### Python
- PEP 8 compliance
- Type hints on all functions
- Docstrings for all public functions / modules
- No hardcoded paths â€” use `pathlib.Path` and environment variables

### TypeScript / React Native
- ESLint + Prettier (check `.eslintrc.json`, `.prettierrc`)
- Strict null checking (`strict: true` in `tsconfig.json`)
- No `any` types without explicit comment
- Component naming: PascalCase
- Variable/function naming: camelCase
- Export screens/services as named exports

### JSON (disease knowledge base, i18n files)
- Organized by disease
- Flat key structure for i18n (e.g., `disease.black_pod.name`, `disease.black_pod.treatment_steps`)
- Verify all 4 languages present before commit
