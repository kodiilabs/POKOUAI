🌱 PokouAI Design System

BECANTY: Beautiful, Efficient, Clear, Accessible, Navigable, Trustworthy, Yielding

Design Philosophy

PokouAI is built for West African smallholder farmers using entry-level Android phones. The design system prioritizes accessibility, clarity, and offline-first thinking.

🎯 Core Design Principles:
  • Beautiful: Agricultural green palette; organic, natural shapes; positive visual feedback
  • Efficient: 3 taps to any core function; large touch targets (48px minimum); offline-responsive
  • Clear: Icons + text; minimal text density; high contrast; multilingual support built-in
  • Accessible: WCAG 2.1 AA compliance; low-literacy friendly; no jargon
  • Navigable: Consistent mental model; tab bar always visible; breadcrumbs on depth > 2
  • Trustworthy: Offline status visible; data never sent without permission; source/tier shown
  • Yielding: Feedback loops (7-day hypothesis card); teaches as it diagnoses

Component Guidelines

Buttons

Primary Button: Large (48px height), forest green (#2d5016), white text, rounded corners, full width on mobile.
Secondary Button: Outlined, 48px height, forest green text, transparent background.
Icon Buttons: 56px squares for large touch targets (essential for outdoor use with gloved hands).
Disabled State: 50% opacity, no pointer events.

Navigation

Bottom Tab Bar: 5 main tabs (Home, Diagnose, Learn, Farm, Settings), always visible, icons + labels.
Back Button: Top-left chevron, returns to previous screen or dismisses modal.
Header: 56px minimum height; title centered, back/menu on sides.

Forms & Input

Text Input: 48px height, 16px font, clear placeholder text.
Selects/Pickers: Native mobile pickers (iOS UIPickerView, Android NumberPicker), 44px tall items.
Photos: Tap to open camera, grid preview, crop interface with guides.
Labels: Always above input, optional indicator if field is optional.

Offline Mode Indicator

Connection Status Bar: Shown below header when offline. Green ("Online") or gray ("Offline"). Status and sync timestamp always visible.

Screen Architecture

The app follows a consistent 3-tier navigation model:

  1. Tab Navigator: Home, Diagnose, Learn, Farm, Settings (bottom tab bar)
  2. Stack Navigator (per tab): Main screen → Detail screens (e.g., HomeScreen → FarmDetail)
  3. Modal Navigator: Modals overlay stacks (e.g., QR code scanner, settings modal)

Screen Map & User Flows

🏠 Home

Quick diagnosis, recent tests, sync status, group mode entry.

📸 Diagnose

Camera → disease detection → treatment steps.

📚 Learn

Disease library, prevention calendar, spaced-rep quiz.

🚜 Farm Log

History of diagnoses, follow-ups, hypothesis tracking.

⚙️ Settings

Language, offline/online toggle, hub connection, about.

👥 Group Mode (tab: home)

Extension worker teaching flow for groups of farmers.

Key Accessibility Features

  • ✅ Minimum 48px touch target height for all interactive elements
  • ✅ Icons paired with text labels (no icon-only buttons)
  • ✅ Color not the only indicator (icons, patterns, text)
  • ✅ High contrast text (WCAG AA: 4.5:1 for normal text)
  • ✅ Multilingual support (English, French, Dioula, Baoulé)
  • ✅ Simple language, no agricultural jargon on first interaction
  • ✅ Offline-first: all core features work without internet
  • ✅ Image-heavy: photos communicate intent where text fails

Responsive Design

App designed mobile-first for 5–6 inch phones (common in rural Ivory Coast). Layouts adapt to:

  • Small phones (320px): 1-column layout, stacked content
  • Standard Android/iOS (360–428px): Primary target, 1-column with occasional side-by-side cards
  • Tablets (600px+): 2-column layout, sidebar navigation option

Dark Mode

Optional dark mode for battery savings on entry-level phones. Uses slightly lighter greens and high-contrast accent colors.
